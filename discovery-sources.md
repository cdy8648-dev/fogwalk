# FogWalk 발견 소스/임계값 확정 스펙 — 권위 목록 기반

> **목적**: 발견요소(랜드마크·지하철·공항 등)를 **런타임 OSM 의존에서 → 권위 있는 공식 목록을 빌드타임에 수확한 frozen 번들**로 전환하기 위한 분류별 소스/임계값 확정표.
> **배경**: OSM 런타임 수집은 (1) 통제 불가(동네 교회 5만 개가 랜드마크로 등장) (2) 커버리지 타이밍 누락(백그라운드 주행 시 잠실역 미발견)의 근본 원인. → [discovery-plan.md](discovery-plan.md)는 기존 OSM 방식 기록.

관련: [[curated-landmarks]] · [[region-pack-download]] · 배터리 헌장([AGENTS.md](AGENTS.md))

---

## 1. 지배 원칙

> **한 분류가 "명확히 통제 가능"한가 = 그 분류에 권위 있는 유한 목록(공식 등록부)이나 강한 notability 신호가 존재하는가.**

- 교회가 막 뜬 이유: 교회는 권위 목록이 없음(전국 5만+). → 카테고리째 넣으면 노이즈.
- 사찰이 통제 가능했던 이유: **전통사찰 지정 목록**(공식 등록부)이 존재.
- **결론**: "분류를 넣는다/뺀다"가 아니라 **"권위 목록/지정에 든 것만 넣는다"**.

빌드타임 수확이 주는 통제력:
```
분류마다 → 소스 선택(공식 등록부 / Wikidata / OSM-gated)
        → 임계값 선택(문화재 지정 / 고도 / notability)
        → 결과 리스트를 사람이 검토·승인
        → frozen 번들로 확정 (런타임은 이 번들만 사용)
```
마지막 **사람 승인**이 최종 품질 게이트. 애매하면 뺀다. 뜨는 것 = 앱에 뜨는 전부 → 예측 가능.

## 2. 발견 분류 확정표 (한국 v1)

### 🟢 Tier 1 — 완벽 통제 (권위 유한 목록 그대로 100% 수집)

| 분류 | 권위 소스 | 임계값 | 규모 | 등급 |
|---|---|---|---|---|
| 지하철/전철 | 국가철도공단·서울교통공사 역사 좌표 (폴백: OSM `station=subway`) | 전 역 | ~1,000 | 거점=희귀 / 그외=일반 |
| 기차역 | 코레일·국가철도공단 역 목록 | KTX·주요역(무배차 간이역 제외) | 수백 | 희귀 |
| 공항 | 한국공항공사 / IATA | 민간공항(군용 제외) | ~15 | 영웅 |
| 국립공원 | 국립공원공단 | 지정 22곳 | 22 | 희귀~영웅 |

### 🟡 Tier 2 — 지정/임계값 게이트로 통제 (여기가 교회·사찰 갈림길)

| 분류 | 게이트(권위 소스) | 통과 | 비고 |
|---|---|---|---|
| 사찰 ⛩️ | **전통사찰 지정 목록**(문체부) | 지정 전통사찰만(~940) | 동네 절 제외 |
| 궁궐·성·유적 🏯 | **국가유산청 국가문화유산포털 Open API** | 국보·보물·사적·명승 지정분 | 무지정 제외 |
| 교회/성당 ⛪ | **문화재 지정된 것만** | 명동성당(사적)·약현성당 등 | 5만 개 → 수십 개 |
| 박물관 🏛️ | Wikidata QID(위키백과 문서 보유) | 국공립·저명 사립 | 동네 전시관 제외 |
| 산·봉우리 ⛰️ | 산림청 100대 명산 / 고도 임계값 | ≥임계 or 명산 | 잡봉 제외 |
| 다리·타워 🗼 | Wikidata / 높이·길이 임계 | 상징적 구조물 | 일반 육교 제외 |

### 🔵 큐레이션 랜드마크 (상징급) — 이미 루틴이 생성 중

| 분류 | 소스 | 규모 |
|---|---|---|
| 전설/영웅/희귀 상징 | Wikidata QID 수동 큐레이션(`constants/curated/*.ts`) | 336곳/9개국 (좌표 추가 필요) |

### 🔴 Tier 3 — 권위 목록 없음 → 자동 포함 금지

- 무지정 교회·동네 절·잡박물관·일반 상점·소규모 관광지·동네 공원 등 롱테일
- **정책**: 카테고리째 제외. 개별로 원하면 큐레이션 목록에 수동 추가만.

### v1 최종 포함/제외
- ✅ 지하철 · 공항 · 기차(주요) · 국립공원 · **지정문화재(궁궐/성/유적/전통사찰/지정교회)** · **QID 큐레이션 랜드마크**
- ❌ 무지정 종교시설 · 잡박물관 · 일반 상점/POI 롱테일

## 3. 해외 확장 (국가별 온디맨드 팩)

권위 소스는 나라마다 다름 → 공통 폴백은 **Wikidata SPARQL(위키백과 문서 보유 = notability)** + **OSM UNESCO/heritage 태그**.

| 소스 | 용도 |
|---|---|
| Wikidata SPARQL | "그 나라의 X 유형 중 위키백과 문서 보유" → 저명 랜드마크·박물관 |
| OSM `heritage`/UNESCO 태그 | 세계유산 자동 인정 |
| 국가별 공식 등록부(있으면) | 일본 문화청·프랑스 Monuments historiques 등 |
| 각국 지하철 GTFS/OSM | 메트로 역사 좌표 |

→ 국가별로 수확·검토 후 `packs/<cc>.json`에 landmarks 추가 (기존 [regionPackDownload.ts](src/services/regionPackDownload.ts) 재사용).

## 4. 데이터 스키마 (frozen 팩에 좌표 추가)

큐레이션의 결정적 결핍 = **좌표**. 모든 항목에 검증된 QID가 있으니 빌드타임 1회 해석:

```
QID → Wikidata P625(좌표) 조회        (상징 랜드마크)
지정번호/역ID → 공식 API 좌표          (문화재·지하철·공항)
→ lat/lng 확정 → frozen 팩에 굽기
```

번들/팩 레코드(압축 튜플 지향, ~30–40B/건 gzip):
```jsonc
// packs/<cc>.json 에 추가되는 배열
"landmarks": [
  { "lat": 37.5796, "lng": 126.9770, "name": "경복궁",
    "cat": "palace", "rarity": "legendary",
    "qid": "Q485238", "src": "heritage:사적",   // 출처 추적(유지보수용)
    "cheer": "…" }
]
```
- `src` = 어느 권위 소스에서 왔는지(문화재 지정번호/역ID/QID) → 재수확·검증 추적성.
- KR은 앱 번들, 해외는 온디맨드(§3).

## 5. 성능 · 유지보수 · 배터리 보증 ⭐

사용자 제약("성능/유지보수/배터리에 이상 없는 선"). 이 설계는 **세 축 모두 현행 이상**:

### 🔋 배터리 — 개선
- **런타임 네트워크 0** — Overpass/Wikidata 호출이 전부 빌드타임으로 이동. 배터리 헌장(핫패스 네트워크 금지)에 완전 부합.
- 발견 판정은 지금과 동일(로컬 인덱스 쿼리). 백그라운드 주행 발열 요인이던 셀별 Overpass가 **완전 제거** → 잠실 누락도 동시 해결.

### ⚡ 성능 — 동일하거나 개선
- **번들 크기**: KR 랜드마크(수천 건) ~수백 KB. 앱은 이미 Mapbox 포함 수십 MB → 체감 0%.
- **시드**: 앱 시작/팩 다운로드 시 **1회** JSON 파싱 + SQLite 트랜잭션 일괄 INSERT(수십 ms). GPS 핫패스 아님 → 헌장 위반 없음.
- **발견 쿼리**: 변경 없음. 이미 미발견분 부분 인덱스 bbox 조회 → 행이 늘어도 근처 몇 개만 반환(SQLite 수백만 행 여유).
- **메모리**: SQLite 백업, JS엔 근처 결과만. 상주 증가 없음.

### 🔧 유지보수 — 개선
- **권위 소스 + 버전드 팩**(`v` 필드) + `src` 출처 추적 → 재수확·감사 가능.
- **사람 검토 게이트** → 품질 예측 가능(뜨는 게 전부).
- **재실행 가능한 수확 스크립트**(`scripts/`) → 문화재 신규 지정·신설역 반영은 스크립트 재실행 + 팩 버전업.
- 매일 루틴은 그대로 활용(큐레이션 선정) + 좌표 수확 스텝만 추가.

## 6. 배포 · 마이그레이션

1. 빌드타임 수확 스크립트로 `packs/kr.json`(+해외)에 `landmarks` 생성.
2. 런타임 발견을 **번들/팩 시드 우선**으로 전환.
3. **OSM 런타임 수집은 플래그로 OFF**(코드 보존) — 만족 확인 후 제거.
4. 마이그레이션: 기존 OSM 자동수집 미발견분 정리(발견 기록 discovered_at 보존), 새 소스로 재시드.

- [ ] `common` 등급을 아예 없앨지(권위 목록만 = 전설/영웅/희귀 + 지하철 일반)
