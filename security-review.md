# FogWalk 보안 점검

작성 2026-06-22. 두 관점: (1) 현재 1.0.0(로컬 퍼스트) (2) Phase 7(백엔드/소셜) 도입 시.

---

## Part 1 — 현재(1.0.0, 로컬 퍼스트) 점검

### ✅ 강점 (구조적으로 공격면이 작음)
- **서버·계정·분석 없음** — 수집/전송/인증 표면이 거의 없음. 모든 데이터 기기 로컬(SQLite + 사진 파일).
- **전송 전구간 HTTPS** — 평문(http) 없음. iOS ATS 준수.
- **SQL 전부 파라미터 바인딩** — 문자열 연결 쿼리 없음 → SQL 인젝션 없음. 동적 컬럼도 코드 화이트리스트.
- **비밀 관리** — `.env`/`*.p8` gitignore + 커밋마다 시크릿 가드 확인. git 추적 안 됨.
- **다운로드 토큰(sk.)은 빌드타임 전용** — 런타임 앱 바이너리에 미포함.
- **렌더 안전** — OSM 이름 등 외부 문자열은 RN `Text`로만 표시(웹뷰/HTML 없음 → XSS 없음). 외부 링크는 하드코딩된 Mapbox/OSM URL뿐.

### ⚠️ 발견 (낮음~중간)
1. **[중간·프라이버시] Overpass 1순위가 `maps.mail.ru`** — 랜드마크 조회 시 **사용자 GPS 좌표가 러시아 mail.ru 운영 미러로** 전송됨. 프라이버시 지향 앱엔 부적절.
   - 권장: 순서를 `overpass-api.de`(공식) 우선으로 바꾸고 mail.ru는 후순위 또는 제거. (장기적으로 자체 호스팅 옵션 고려)
2. **[중간] Mapbox 퍼블릭 토큰(pk.)이 앱 바이너리에 임베드** — 정상 동작이지만 추출 가능.
   - 권장: 해당 토큰이 **public 스코프만**인지 확인(시크릿 스코프면 즉시 회전), Mapbox 대시보드에서 **URL/앱 제한** + **사용량 알림** 설정(유출 시 과금 abuse 방지).
3. **[낮음] 로컬 데이터 추가 암호화 없음** — 위치 이력·사진이 앱 샌드박스에 평문. iOS 기기 암호화(패스코드)+샌드박스에 의존. 로컬 전용이라 위험 낮음.
   - 선택: 민감도 높이려면 위치 이력 테이블에 `NSFileProtection`(complete) 적용 검토.
4. **[낮음] Overpass로의 좌표 전송 자체가 프라이버시 노출** — 미러 운영자가 대략 위치 관찰 가능. 개인정보처리방침에 OSM 전송은 명시돼 있음(OK). 1번 변경과 함께 다루면 됨.

> 결론: **로컬 퍼스트라 현재 심각한 보안 취약점 없음.** 실질 액션은 **#1(mail.ru 엔드포인트 정리)** 하나 + **#2(Mapbox 토큰 스코프/제한 확인)** 권장.

## Part 2 — Phase 7(백엔드/소셜) 도입 시 보안 과제

서버·계정·UGC·사진/위치 공유가 생기면 **공격면이 근본적으로 커짐.** 진입 전 반드시 설계.

1. **위치 프라이버시 (최우선 위험)** — 사진·발견·위치를 서버/타인과 공유하면 집·동선 노출 → 스토킹 위험.
   - 좌표 **퍼징(그리드 스냅/지터)**, **opt-in 공유**, 사적 장소 정밀좌표 미노출, "최근 위치/실시간" 금지, 집 추정 방지.
2. **Supabase RLS (가장 흔한 사고)** — anon 키는 앱에 공개되므로 **Row-Level Security가 유일한 방어선.** 모든 테이블 RLS 필수(본인 것만 수정/공개분만 읽기). **`service_role` 키는 절대 앱에 넣지 말 것**(서버 전용).
3. **EXIF GPS 제거 (놓치기 쉬움)** — 업로드 사진에 GPS EXIF가 남으면 앱에서 좌표를 퍼징해도 **사진 메타데이터로 정확 위치 유출.** 업로드 전 EXIF 스트립 필수.
4. **인증/세션** — Apple Sign In, 토큰은 **Keychain(expo-secure-store)** 보관(SQLite/AsyncStorage X), 리프레시·로그아웃·만료 무효화.
5. **UGC 모더레이션 (Apple Guideline 1.2 필수)** — 신고/차단, 0-tolerance EULA, 신고 **24h 내 대응**, 콘텐츠·악성유저 제거, CSAM/불법 콘텐츠 대응 파이프라인. 없으면 리젝.
6. **Storage 접근정책** — 버킷 RLS/서명 URL, 열거(enumeration) 방지.
7. **API 남용/스크래핑** — "주변 공개 사진/발견" 읽기 엔드포인트는 rate limit·페이지네이션·퍼징·인증 요구(전수 수집 방지).
8. **컴플라이언스 전면 갱신** — 이제 데이터 수집·전송·연결됨 → **App Privacy 라벨 대폭 수정**, 개인정보처리방침 개정, **앱 내 계정/데이터 삭제**(Apple 필수), GDPR/CCPA export/delete, 보존정책.
9. **서버 인젝션** — PostGIS RPC/함수/쿼리 전부 파라미터화, 입력 검증.
10. **위치 스푸핑/신뢰** — 소셜·랭킹이 생기면 GPS 스푸핑으로 가짜 발견/게시 가능 → 서버측 타당성(속도/연속성) 검증.
11. **로그/관측성** — 서버·클라 로그에 좌표·PII 남기지 않기.

> Phase 7은 "보안이 곧 기능"인 단계. 시작 전 **위협 모델 + 데이터 분류(공개/비공개) + RLS 정책** 설계 문서부터 쓰는 걸 강력 권장.
