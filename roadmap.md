# FogWalk 출시 이후 로드맵

> 1.0.0(현재 심사 중)이 승인된 뒤의 작업을 phase로 정리. 위에서 아래로 권장 순서.
> 표기: 🔁 새 빌드 필요(네이티브/설정 변경) · 📝 JS-only(빌드 불필요로 검증 가능) · ⚠️ App Privacy 라벨 영향

---

## Phase 4 — 1.0.1 빠른 후속 + 안정화  (작게, 바로)
출시 직후 실사용 신호를 보면서 빠르게 다듬는 단계.

- **아이보리 맵 반영** 🔁 — 이미 코드 완료(`22e9aec`). 다음 production 빌드에 포함, 스크린샷도 아이보리로 교체.
- **출시 후 핫픽스** — 실제 사용 중 발견되는 버그/이상 대응 (최우선).
- **소소한 폴리시** 📝 — 도트그리드 배경, Profile/Map에 핀보드 톤 마저 확장.
- **그라데이션 복원** 🔁 — `expo-linear-gradient` 추가해 정체성 아바타·여권 카드 그라데이션(현재 단색 대체분).
- **빌드/툴 위생** 🔁 — Mapbox 다운로드 토큰 플러그인 prop → `RNMAPBOX_MAPS_DOWNLOAD_TOKEN` env 마이그레이션(현재 deprecated 경고), Node LTS 업그레이드(≥20.19.4).

## Phase 5 — 성능 & 견고성  (사용자·데이터 늘기 전에)
지금은 문제없지만 밝힌 땅/마커가 쌓이면 체감되는 것들. 감사에서 식별됨.

- **FogLayer 최적화** 📝 — `cellsToMultiPolygon`를 near/far 2번 전체 타일에 매번 → 증분/캐싱. 최대 확장성 병목.
- **마커 뷰포트 컬링** 📝 — 랜드마크·사진·지하철을 화면 영역만 렌더(현재 전부 렌더).
- **순수 유틸 단위 테스트** 📝 — h3/distance/xp/calendar/milestones. 회귀 방지(이번 Overpass류 사고 예방), CI 기반.
- **백그라운드 축하 큐 정리** 📝 — 복귀 시 토스트 몰림 방지(상한/병합).
- (선택) **db.ts 도메인 분리** 📝 — 파일 비대.

## Phase 6 — 리텐션 기능  (솔로 앱 강화 · 백엔드 불필요)
재방문·습관화를 돕는 기능. 대부분 네이티브 권한이라 빌드 필요.

- **데일리 산책 알림** 🔁⚠️ — `expo-notifications`. 시간 설정(settingsStore), 스트릭 유지 유도. (로컬 알림은 GPS 자동실행 불가 — "열기" 탭 유도) ⚠️ 알림 권한.
- **발견 시네마틱 연출** 📝 — 랜드마크 발견/레벨업 시 카메라 줌·리플레이 등 보상감 강화.
- **지하 구간 reveal** 📝 — 역↔역 점프 시 노선 따라 보간 reveal(지하철 발견과 연계).
- (선택, 큼) **위젯/잠금화면** 🔁 — 오늘 거리·스트릭 위젯.
- (선택, 큼) **HealthKit/Apple Watch 경로 백필** 🔁⚠️ — 기존 운동 경로를 탐험으로 가져오기. 네이티브 모듈 + 건강 권한.

## Phase 7 — 소셜  (가장 큰 단계 · 백엔드 도입 = 아키텍처 전환점)
"로컬 퍼스트 → 서버 동기화"로 넘어가는 큰 결정. 시작 전 별도 설계 필요.

- **데이터 추상화** 📝 — `db.ts` 위에 repository/데이터소스 계층 → 서버 동기화 대비.
- **백엔드** 🔁 — Supabase/PostGIS (인증·저장소·공간쿼리).
- **Apple Sign In** 🔁⚠️ — 계정 도입.
- **소셜 흐름** 🔁⚠️ — 주변 공개 사진 익명 읽기 → 공유 업로드 → 좋아요.
- **모더레이션 + 프라이버시** 🔁⚠️ — 신고/차단/UGC 검수(Apple UGC 필수), 위치 퍼징/옵트인.
- ⚠️ **App Privacy 라벨 전면 갱신** — 이제 데이터가 서버로 수집·전송됨 → 현재 "위치만, 미연결" 신고를 대폭 수정해야 함. 약관/개인정보처리방침도 개정.

## 운영(상시)
- 심사 노트(백그라운드 위치 사유) + 데모 영상은 향후 제출마다 Notes에 유지.
- 새 권한/SDK 추가 시 **App Privacy 라벨·권한 문구·개인정보처리방침** 동시 점검.
- 큐레이션 랜드마크 목록(`curatedLandmarks.ts`)·마일스톤(`milestones.ts`) 값은 피드백 보며 튜닝.

## 의존성 메모
- Phase 4~6은 서로 독립적 — 우선순위대로 골라서 진행 가능.
- Phase 7은 진입 시 4~6의 "데이터 추상화"가 선행되면 수월. 시작 전 설계 문서부터.
